## 方向 A：Agent 测试与评估（重点探索）

### 概述

为 AI Agent 构建系统化、自动化的测试和评估体系。这个方向的核心挑战是：**LLM 输出是非确定性的，传统 assert 不再适用**。

### A1：评估指标设计

**学习目标**：为 Agent 设计多维度评估指标

**操作建议**：
1. 研究以下评估维度：
   - **Answer Relevance**：回答是否解决了用户问题？
   - **Faithfulness**：回答是否忠于参考资料（无幻觉）？
   - **Tool Use Correctness**：工具调用是否准确（选对工具、参数正确）？
   - **Safety**：是否泄露敏感信息？是否产生有害内容？
   - **Latency**：完成一次任务的耗时
   - **Cost**：token 消耗和费用
2. 学习现有评估框架：
   - **Ragas**（`pip install ragas`）：RAG 评估框架
   - **DeepEval**（`pip install deepeval`）：LLM 应用测试框架
   - **LangSmith**：LangChain 官方的追踪和评估平台
3. 为第四阶段的项目设计一套评估指标

**Milestone A1**：
- [ ] 列出你项目的 5+ 个评估指标及其定义
- [ ] 用 Ragas 或 DeepEval 跑通一个评估示例
- [ ] 能解释 LLM-as-judge 的优缺点

### A2：测试数据集构建

**学习目标**：构建有代表性的测试数据集

**操作建议**：
1. 收集/构建测试数据集：
   - 从真实用户场景中提取 50+ 个测试用例
   - 覆盖正常情况、边缘情况、对抗性输入（prompt injection）
2. 为每个用例标注：
   - 预期行为（应该做什么）
   - 预期输出特征（包含什么、不包含什么）
   - 难度等级（简单 / 中等 / 困难）
3. 了解数据集管理工具（DVC、Git LFS）

**Milestone A2**：
- [ ] 50+ 个测试用例的数据集，每个都有预期行为标注
- [ ] 数据集按场景分类组织
- [ ] 有数据集版本管理方案

### A3：自动化评估流水线

**学习目标**：把评估集成到 CI/CD

**操作建议**：
1. 用 GitHub Actions 或本地脚本搭建：
   - 代码变更 → 触发评估 → 生成评分报告
   - 低于阈值 → 标记为 fail
2. 报告包含：
   - 各维度得分
   - 与前一次评估的对比（趋势图）
   - 失败的 case 详情
3. 学习 LangSmith 的 trace 功能：追踪每次 Agent 调用的完整过程

**Milestone A3**：
- [ ] 自动化评估流水线能跑通
- [ ] 有评分报告（哪怕只是文本输出）
- [ ] 能展示两次评估之间的趋势对比

### A4：Agent 行为监控与调试

**学习目标**：构建 Agent 执行过程的可视化和调试能力

**操作建议**：
1. 了解 Agent 调试的挑战：
   - 每次运行可能不同
   - 多步任务的中间状态难以定位
   - LLM 的"想法"不容易追溯
2. 实现简单的 trace 系统：
   - 记录每一步的 Thought、Action、Observation
   - 记录工具调用的参数和结果
   - 记录 token 消耗和时间
3. 尝试 LangSmith / LangFuse 等开源追踪平台

**Milestone A4**：
- [ ] 能重现一次 Agent 运行的完整执行轨迹
- [ ] 能定位出"在哪一步开始出错"
- [ ] 了解至少一个 Agent 监控/追踪平台

## 综合探索项目

不管你最终选择哪个方向，都做以下项目来综合前五个阶段的成果：

### 项目：Agent 质量评估平台

构建一个平台，能对 AI Agent 进行系统化评估：

1. **数据集管理**：上传、编辑、版本管理测试用例

## 常见陷阱

- **只学不练**：AI 领域更新太快，最快的学习方式是动手做项目
- **追新焦虑**：新的框架和工具层出不穷，掌握底层原理比追新重要
- **不写文档**：你学到的东西值得记录下来，方便回顾也方便分享
- **忽略安全**：Agent 调用工具、访问数据，必须有安全意识
